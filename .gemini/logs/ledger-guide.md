# 📜 에이전트 위임 장부 가이드 (Agent Delegation Ledger Guide)

이 문서는 에이전트 위임 장부의 목적과 기록 방식에 대해 설명합니다.

### 🎯 목적
모든 에이전트 간의 호출 흐름, 지시 내용, 그리고 그 결과를 기록하여 프로젝트의 투명성과 추적성을 보장합니다.

### 📏 기록 규칙
- **파일명**: `src/logs/agent-ledger.md`
- **기록 시점**: 서브에이전트 호출 직전(대기중) 및 호출 완료 직후(결과 반영).
- **시각 포맷**: `YYYY-MM-DD HH:mm:ss` (반드시 정확한 시스템 시각을 조회하여 기록).
- **지시 요약**: 에이전트에게 전달된 핵심 명령어를 한 줄로 요약하여 기록.
- **토큰 사용량 표시(중요)**: 에이전트가 실제 사용한 토큰 정보를 반드시 기록.

### 📋 장부 데이터 구조
| 컬럼명 | 설명 |
| :--- | :--- |
| 시간 | 호출 또는 완료된 정확한 시스템 시각 |
| 호출자 | 작업을 지시한 에이전트 이름 (예: orchestrator) |
| 수신자 | 작업을 수행하는 에이전트 이름 (예: frontend) |
| 태스크 ID | 해당 작업이 속한 Task ID (예: TASK-001) |
| 호출 지시 내용 | 에이전트에게 내린 명령의 핵심 요약 |
| 결과 | 작업 상태 (대기중, success, failed 등) |
| 토큰 (P/C/T) | 사용된 토큰 정보 (Prompt / Completion / Total) |

### 💡 토큰 기록 팁
- 에이전트 실행 완료 후 터미널에 출력되는 **Usage Stats**를 확인하세요.
- 기록 형식: `P: [프롬프트] / C: [완성] / T: [합계]` (예: `P: 500 / C: 120 / T: 620`)
- 백그라운드 작업의 경우, 호출 결과 보고 시점에 터미널 로그에서 수치를 추출하여 기입합니다.

---

### 🏁 최종 작업 보고 (Reporting)
모든 에이전트 작업이 종료되면 마지막으로 작업을 수행한 에이전트(또는 오케스트레이터)는 장부 데이터를 기반으로 **최종 토큰 사용 보고서**(`src/logs/token-usage-report.md`)를 작성해야 합니다.

1.  **데이터 합산:** `agent-ledger.md`의 모든 토큰 정보를 합산하여 총 사용량을 계산합니다.
2.  **효율성 분석:** 에이전트별 평균 소모량을 계산하여 비용 효율성을 점검합니다.
3.  **보고서 확정:** 작성된 보고서를 프로젝트 루트 또는 `src/logs/`에 안전하게 보관합니다.
